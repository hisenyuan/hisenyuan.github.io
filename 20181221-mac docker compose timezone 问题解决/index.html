mac docker compose timezone 问题解决

Posted on 2018-12-21 Edited on 2025-07-19 In docker

一、背景介绍

想用docker搭建一个lnmp环境

使用这个脚本：

执行命令之后报错

二、报错原因

  1. mac机器的时间路径与linux不一样
  2. lnmp.yml redis用到了这个时间

三、解决办法

  1. 查看当前机器时间文件真实位置(/etc/localtime 这个路径是不让共享给docker的)
  1. 设置位置 ：Docker -> Preferences… -> File Sharing
  2. 新增共享目录：/var/db/timezone
  3. apply & restart
  4. 修改lnmp.yml配置文件 全局替换：/usr/share/zoneinfo/Asia/Shanghai 为：/var/db/timezone/zoneinfo/Asia/Shanghai
