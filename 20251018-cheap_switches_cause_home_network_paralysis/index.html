廉价交换机导致家庭网络瘫痪半年

Posted on 2025-10-18 In Tech Symbols count in article: 808 Reading time ≈ 1 mins.

0. 总结

故障开始：2025年4月13日
故障结束：2025年10月12日
故障原因：交换机光模块发热导致断流

  • 发现问题后使用猫扇对着吹，2天没都正常

交换机：keepLINK2.5G交换机(5口2.5G+1口万兆光口)。( 劝退，温度稍微高点就不行 )

1. 网络情况

进：入户光纤—弱电箱—顶柜(预计放网络设备)—吊顶(为了不走明线)—机柜(家政柜改的)—路由器
出：路由器—交换机(SFP+)—弱电箱—交换机(SFP+)—内网设备(AC+AP)

规划不到位+新需求变更导致的畸形解决方案.

2. 排查过程

刚开始连接上网络(本地+WIFI)，DHCP IP 分配正常，有时候可以上网，有时候不行。
没有排查到问题，由于家里有闲置的流量卡，就优先用热点了。

期间由于工作上事情比较多，周末又有安排或者不想动，反正就是没动力去查…

于是就导致家里的 IOT 设备失效…因为都可以手动操作也就没什么影响。

然后由于联通基站问题，这边流量也不太快，联通上门查了反馈的。
投诉也没用，就说等安排扩容基站，赔了一个月套餐费。

期间家里领导也断断续续提过快点解决，
由于之前尝试的办法都不行，我一度怀疑是家里网络搞太复杂导致的。

国庆节后，不那么忙，就想着回到最原始的办法，挨个设备排查。

  1. 用 10m 的网线，把路由器和电脑直接连接：正常
  2. 用 10m 的网线，把路由器和弱电箱交换机连接：正常
  3. 发现光纤口灯不亮：找到问题
  4. 怀疑弱电箱交换机坏了，京东买个同型号：判断错误
  5. 怀疑机柜里交换机坏了，替换新交换机：可以
  6. 换机柜中交换机过程中，发现光模块太热，会不会是发热导致？
  7. 安排风扇降温：多天正常

总结：交换机光模块太热导致罢工

3. 解决问题

应急：安排风扇
常规：购买 TP-Link 2.5G+万兆 交换机替换

  • 目前移除散热风扇，情况正常

4. 感悟

遇到问题，排除法还是好用。

不要懒，不然吃亏的还是自己。
网络瘫痪期间

  • 用联通的热点上网苦不堪言，还好可以看书或者别的…
  • IOT 设备罢工，感应开关灯之类的没法用，稍微有点影响体验
